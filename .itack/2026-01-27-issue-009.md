# Update worktree-itack plugin to link ccx session to itack issue

After starting a ccx session for an itack issue, call `itack set-session` to record which session is working on the issue.

**Requirements:**
1. Capture the ccx session name from `ccx start` output
2. Call `itack set-session <issue-number> <session-name>` after starting

**Updated workflow step 3:**
```bash
# Start session and capture its name
session=$(ccx start --cwd "${WORKTREE}/..." "prompt..." 2>&1 | grep -oP 'ccx-[a-f0-9]+')

# Link the session to the issue
itack set-session <issue-number> "$session"
```
